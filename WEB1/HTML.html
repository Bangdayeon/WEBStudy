HTML

Hyper Text Markup Language
*HyperText = 태그를 의미한다

- 태그는 대부분 두쌍씩 사용한다
- 태그는 중첩하여 사용이 가능하다 (부모태그&자식태그)
- 앞에 있는 태그(<>)는 열리는 태그, 뒤에 있는 태그(</>)는 닫히는 태그

Document metadata

* 페이지 정보를 포함하는 태그들
  1. <head>
    * 본문 관련 설명을 입력
    * <title>, <meta>와 같은 아래 다른 코드들을 넣는다
  2. <meta>
    * 해당 문서의 메타데이터를 정의
    ex) <meta charset="utf-8">
    ex) <meta name="viewport" content="width=device-width">
  3. <title>
    * 웹페이지의 제목 지정
    * 브라우저에서 웹페이지를 검색할 때 사용한다
    * 지정하지 않으면 파일명을 그대로 사용
  4. <base>
    * 해당 문서의 베이스가 되는 URL을 사용할 경우 이용하는 태그
    * 문서에서 한 번만 사용할 수 있다
  5. <link>
    * 해당 문서와 관련된 문서 및 외부 소스를 링크하는 태그
    * css나 외부 리소스 아이콘을 사용하는 등의 경우에 사용
    ex) <link rel="stylesheet" href="css/master.css">
    ex) <link rel="icon" href="favicon.ico">
  6. <style>
    * 문서에 적용되는 스타일 정보를 포함하는 태그
    * 내용에 CSS가 적힌다

위 태그들을 사용하여 아래와 같은 형태의 html 문서의 기본 구조를 만들 수 있다
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width">
    <title> @@@ </title>
  </head>
  <body>
    somethings
  </body>
</html>

Sectioning root

<body>
* 본문 내용 관련 태그들을 넣는 태그
* 사용자에게 보여지는 내용을 작성한다

Content sectioning

* 컨텐츠의 논리적 개념과 관련된 태그들
* 화면 구조를 구성하기 위한 태그들로, 사용자에게 직접 보여지지 않는다
* a.k.a. Box Tags
  1. <address>
    * 정보를 통한 링크들을 표현하는 태그
    * box level
    ex) 메일, 번호 등
    dybang00@gmail.com
    (010)1234-5678
  2. <article>
    * 재사용이 가능한 의미 있는 컨텐츠 묶음을 표현는 태그
    * box level
  3. <aside>
    * 페이지의 기본 컨텐츠와 간접적으로 관련된 내용을 표현하는 태그
    * sidear나 call-out box와 같은 형태로 주로 사용
    * box level
  4. <footer>
    * 상위 컨텐츠 요소에 대한 바닥글을 표현하는 태그
    * box level
  5. <header>
    * 일반적으로 소개 및 검색을 위한 컨텐츠를 표현하는 태그
    * box level
  6. <h1>~<h6>
    * 6단계로 나누어 제목을 표시하는 태그
    * 숫자가 커질수록 표시 크기가 작아진다
  7. <hgroup>
    * 제목 하위에 포함되는 내용을 담은 컨텐츠를 표현하는 태그
    * box level
  8. <main>
    * 문서의 중심이 되는 컨텐츠를 넣는 태그
    * box level
  9. <nav>
    * 다른 페이지에 대한 탐색 링크를 제공하는 컨텐츠를 표현하는 태그
    * box level
    ex) 메뉴, 목차, 색인 등
  10. <section>
    * 페이지 내의 box 요소 중 가장 구체적인 컨텐츠를 표현하는 태그
    * 항상 제목이 있어야 한다

    * 설명 및 정의를 표현하는 태그
    * dl태그(설명목록) 내에서 dt태그(용어)에 대한 설명 및 정의를 표현한다

Inline text semantics

  1. <a>
    * 링크를 삽입하는 태그
    * href 속성 : 링크 주소를 지정한다(필수)
    * download 속성 : URL을 다운로드 한다
    * target 속성 : URL 페이지를 어떻게 띄울 것인지 지정한다
    속성값으로, "_self" : 현재 페이지(Default)
    "_blank" : 새로운 탭
    "_parent" : 현재의 부모 브라우저에 띄운다. 없으면 self (뭔소리임)
    "_top" : 가장 조상인 브라우저에 띄운다. 없으면 self (뭔소리임22)
    op.gg의 링크
  2. <abbr>
    * 약자를 나타낸다
    BDY
  3. <b>, <strong>
    * 글자의 굵기를 굵게 한다
    * b : 중요도가 높아지는 건 아니고 걍 굵게만
    * strong : 중요도도 같이 높아짐!
  4. <bdi>
    * 태그 내부의 텍스트를 주변 텍스트와 분리하여 처리한다
    * 웹사이트에서 일부 텍스트에 대한 동적인 동작을 해야할 때에 유용(먼소리~)
    방<-다연('방'에 bdi를 씌워봤답니다~~)
  5. <bdo>
    * 텍스트의 현재 방향을 재정의하여 텍스트가 다른 방향으로 렌더링되도록 한다
    * 뭔소리인지 모르겠습니다
  6. <br>
    * 줄넘김을 하는 태그
    * 열림닫힘이 아니라 한짝만 쓰는 태그이다
  7. <code>
    * 코드를 표현하는 태그
    * CSS를 사용해서 보기좋은 형식으로 바꿀 수 있다

    * 포함된 컨텐츠를 data로써 정의한다(사실 잘 모르겠음)

ITEM TAGS

- 사용자에게 보여지는 것들을 표현하기 위한 태그
- block/inline level로 나누어진다
  * block : 한 줄에 하나
  * inline : 공간이 허용되면 다른 태그 옆에 배치
- content : tag 내부에 들어간 내용
- Element : Opening Tag - Content - Closing Tag까지
- Attributes(속성) : opening tag에서 태그에 대한 자세한 속성값을 설정
  1. <a>
    - 링크(anchor)
    - ex) <a href="링크 주소" target="_blank" title="~~">
    ㄴ href(HyperText Reference)
    ㄴ target="_blank" : 링크 클릭 시, 새 창에서 페이지가 열리게 하는 속성
    ㄴ title="~~" : 커서 올렸을 때, 링크가 어떤 내용을 담고 있는지 툴팁으로 보여주는 기능
