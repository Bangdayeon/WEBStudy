Clean Code JS

js 코드를 깔끔하게 작성하는 법

변수

var의 사용을 지양하기
var : "함수" 스코프를 가짐
=> 같은 main 함수 내에서의 정의라면 분기문 내의 변경도 분기문 바깥까지 영향을 끼칠 수 있다
let, const : "블록 단위" 스코프 및 TDZ(Temporal Dead Zone) 속성을 가짐 -> 더 안전하게 코드를 작성할 수 있다
=> 분기문(블록) 내의 변경이 분기문 바깥에 영향을 끼치지 않는다

ex)
var name = '이름';
var name = '이름2';
=> 재할당과 재선언이 가능함

let name = '이름'
let name = '이름2'(x)
name = '이름2' (o)
=> 재선언이 불가능함

const name = '이름'
const name = '이름2' (x)
name = '이름2' (x)
=> 재할당 및 재선언이 불가능함

이런 식으로 const를 최대한 사용하고 값도 바꿀 수 있음
: 재할당이 아닌 값만 바꾼 것이기 때문에 문제 없다
